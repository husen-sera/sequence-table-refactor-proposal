§3 · What we propose — refactor

One job per file

🧱 Layout

Column widths, padding, alignment — where "white space" gets fixed by design

📜 Scrolling

How the long list moves and remembers its place

⏳ Loading

When to show or hide the loading overlay

🃏 Cards

How each card displays inside a cell

🗂 Grouping & rarities

Expand / collapse, rarity badges

📌 Sticky elements

Row numbers, back-to-top buttons

🔀 Built beside the old version, behind a switch. When ready, we flip the switch. If anything breaks — flip it back instantly, no emergency deploy.
§4 · Why refactor — three concrete payoffs

Organized, faster to debug, easier to extend

1 · Organized, not gigantic

Understand one ~300-line file in ~30 min vs. ~12 hours reading the giant file today.

2 · Faster root cause

A "white space" report goes straight to the Layout file. First-attempt fix rate: ~10% → ~70%.

3 · Easier features & debugging

Change one file, leave others untouched. Each file has its own small tests — we know what broke before the user does.

§7 · The honest framing

When is each side right?

Patch — if you believe the next attempt has a 90% chance

Then refactoring is over-engineering. We should just patch it.

Refactor — 22 prior failures suggest the opposite

The next patch is no more likely than the last. The structure of the code is the obstacle, not the developer's skill.

🔍 The indirect benefit: even if the refactor doesn't directly find the root cause, it makes it possible to find. The search space shrinks from 4,778 lines → ~300.

§11–12 · Safety levers & risks

Every safety lever is manual — the team stays in control

🧯 When something goes wrong

Prod regression → flip the switch off, users see the old version instantly. No 2 AM page.
Parity bug in W5 → delay cutover 2–3 days, don't ship until clean.
Mid-refactor feature request → backlog it. Scope is not bent.

⚠️ Top risks, handled

Hidden complexity → Week-1 spike.
Slippage → Week-7 buffer pre-approved.
Scope creep → strict "parity only".
Old code lingers → deletion in W6 is a gate, not optional.

🚦 Protecting against scope creep is the PM's primary job for these 6 weeks — it's the most common cause of refactor-project failure.
